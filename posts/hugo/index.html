hugo+Stack主题搭建

Jan 29, 2022
2 minute read

部署Hugo环境

Linux

hugo

1

yum install hugo

OR

1

apt-get install hugo

git

1

yum install git

OR

1

apt-get install git

Windows

Hugo下载地址

https://github.com/gohugoio/hugo/releases

注意：win用户下载hugo_extended

解压到喜欢的目录里把目录添加到环境变量中

打开命令行（cmd/powerShell）输入hugo version显示表示安装成功

git下载地址

Git - Downloading Package

安装完就行啦

开始搭建

本地搭建

打开命令行（cmd/powerShell）输入

1

hugo new site 你喜欢的网站名

hugo会给你生成一个网站

本blog主题是Stack(文档)

或者你也可以到Hugo Themes下载你喜欢的主题

1

cd 你设置的网站名/themes/

把主题放到这个目录下

或者通过git下载

1

git clone 你喜欢的主题的git链接

配置Stack

  1. 把 你设置的网站名/themes/hugo-theme-stack/exampleSite/下的config.yaml复制到网站的根目录（指你设置的网站名）下

  2. 删除网站根目录下的config.toml

  3. 打开config.yaml添加相关的内容，参考官方文档(上面有链接)

本地运行

在cmd/powerShell切换到网站根目录下输入

1

hugo server

然后在浏览器中输入http://127.0.0.1:1313/就可以访问网站啦

编写博客

文章采用MarkDown语法编写

可以用MarkText工具来编写Blog或者使用在线的编辑器编写

文章目录结构

  • 文章存放在：网站根目录/content/posts/

  • 分类存放在：网站干目录/content/categories/

尝试编写第一个文章

在命令行中输入

1

当然 也可以自己手动创建= =

修改左侧菜单

https://docs.stack.jimmycai.com/zh/configuration/custom-menu

文档中第二种方法：

若在- identifier前添加social关键字会让图标后面的字隐藏

social:
    - identifier: github
      name: GitHub
      url: https://github.com/
      params:
          icon: brand-github
          newTab: true

去除右侧categories

修改config.yaml

widgets:
    homepage:
        - type: search
        - type: archives
          params:
              limit: 5
        # - type: categories
        #   params:
        #       limit: 10

部署到github上

创建GitHub项目

首先得注册一个GitHub账号，由于某些原因GitHub可能无法访问（搭梯子

创建一个名为你的Github用户名.github.io的项目

在网站根目录下打开命令行输入

1

hugo --theme=你下载的主题文件夹名（网站根目录/themes/下的那个）

这条命令会将项目文件放在public目录下，进入public目录 cd/public

提交GItHub项目

在终端输入

1
2
3
4
5

git init
git add -A
git commit -m "注释"
git remote add origin https://github.com/你的Github用户名/你的Github用户名.github.io.git
git push -u origin master

GitHub已不支持通过账号密码来提交文件，需要申请一个token

申请token方法

  1. 网页端登陆GitHub

  2. 右上角头像小箭头

  3. Settings-> Developer settings-> Personal access tokens

  4. 点击Generate new token

  5. 如果不知道勾啥全打勾就行了（

发布完成

访问：https://你的Github用户名.github.io/ 显示正常页面表示成功部署到github上
